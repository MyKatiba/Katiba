# Signing Keys & Fingerprints

This document outlines how to retrieve the SHA-1 fingerprints for both debug and release keystores used in the Katiba project.

## Prerequisites

- Ensure Java Development Kit (JDK) is installed.
- Ensure `keytool` is available in your system path or point to it directly (e.g., `"C:\Program Files\Java\jdk-21\bin\keytool.exe"`).

## Debug Certificate

The debug certificate is automatically generated by Android Studio.

**Keystore Location:** `%USERPROFILE%\.android\debug.keystore`
**Alias:** `androiddebugkey`
**Password:** `android`

### Command to Generate SHA-1 Fingerprint

Run the following command in PowerShell:

```powershell
keytool -list -v \
-alias androiddebugkey \
-keystore "$env:USERPROFILE\.android\debug.keystore" \
-storepass android
```

or if `keytool` is not in your PATH:

```powershell
& "C:\Program Files\Java\jdk-21\bin\keytool.exe" -list -v -alias androiddebugkey -keystore "$env:USERPROFILE\.android\debug.keystore" -storepass android
```

## Release Certificate

The release certificate is used for building the production version of the app.

**Keystore Location:** `%USERPROFILE%\.android\release-key.jks`
**Alias:** `katiba-release-key`
**Password:** *(Prompted during execution)*

### Command to Generate SHA-1 Fingerprint

Run the following command:

```powershell
keytool -list -v \
-keystore "$env:USERPROFILE\.android\release-key.jks" \
-alias katiba-release-key
```

or with full path to `keytool`:

```powershell
& "C:\Program Files\Java\jdk-21\bin\keytool.exe" -list -v -keystore "$env:USERPROFILE\.android\release-key.jks" -alias katiba-release-key
```

### Stored Fingerprints

For reference, the generated fingerprints have been saved locally at:
- Debug: `%USERPROFILE%\.android\katiba-debug-fingerprint.txt`
- Release: `%USERPROFILE%\.android\katiba-release-fingerprint.txt`
